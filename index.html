Transcripción Brechas

¡Saludos!


Les doy la bienvenida al módulo 1 con el tema “breaches”.


Para comenzar, es importante aclarar este concepto. Las brechas de seguridad son incidentes que permiten a personas no autorizadas el acceso a información, también se les conoce como “fugas de información”.


A continuación, les voy a comentar algunas de las posibles causas.

Estas, se dividen en 2 tipos: organizativas y técnicas. 


Dentro de las organizativas se encuentran las siguientes:

  • Falta de clasificación de la información

  • Falta de conocimiento

  • Ausencia de procedimientos

  • Falta de acuerdos de confidencialidad


Dentro de las causas técnicas, se puede mencionar:

  • Código malicioso o malware

  • Acceso no autorizado a sistemas e infraestructuras

  • Generalización del uso de almacenamiento en la nube para cualquier tipo de información

  • Uso de tecnologías móviles para el trabajo diario


Las consecuencias de este tipo de ataques pueden ser muy variadas, algunas de ellas son:

  • Daño a la imagen

  • Consecuencias económicas

  • Consecuencias legales 

  • Otras consecuencias: En esta categoría pueden entrar consecuencias institucionales, gubernamentales, entre otras.


Por último, se expondrán algunas ideas para reducir brechas y costos:

  • “Nunca confíe, siempre verifique”.

  • Cifrado de datos

  • Herramientas de monitoreo

  • Educación en ciberseguridad

  • Planes de respuesta
